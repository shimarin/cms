# ADR-001: トップレベルdocs/をデフォルトvhost兼フォールバックとして使う

## 状況

当初は `vhosts/default/` をフォールバックvhostとして使う設計だった。
vhostを使わない単純な運用では `vhosts/default/docs/` という深いパスにコンテンツを置く必要があり、手間だった。

## 決定

トップレベルの `docs/` と `templates/` をデフォルトvhost兼フォールバックとして使う。

- vhostなし運用: `docs/` にコンテンツを置くだけでよい
- vhost運用: vhost固有コンテンツは `vhosts/<hostname>/docs/` に、全vhost共有リソース（画像等）はトップレベル `docs/` に置く

ファイル検索はvhost固有 → トップレベルの順でフォールバックする。
テンプレートも同様にvhost固有 → トップレベルの順。

## 理由

- vhostなし運用のセットアップを極力シンプルにしたかった
- `templates/` はもともとグローバル共有用だったため、トップレベルに置くのが自然

## トレードオフ

- トップレベルに置いたものは全vhostから見えるため、「vhostを運用する場合はトップレベルには共有してよいリソースしか置かない」というルールを運用側が守る必要がある
