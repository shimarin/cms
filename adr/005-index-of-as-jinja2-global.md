# ADR-005: 記事一覧取得をJinja2グローバル関数index_ofとして提供する

## 状況

ブログ記事のインデックスページやトップページの「最新記事」表示のために、
ディレクトリをスキャンして記事一覧を取得する仕組みが必要。

## 選択肢

**A) フロントマターで宣言する方式**
```yaml
index_of: /blog
```
サーバー側がフロントマターを見て自動的にスキャン結果を変数として渡す。

**B) Jinja2グローバル関数として提供する**
```jinja2
{% for article in index_of("/blog") %}...{% endfor %}
```
テンプレート側から能動的に呼び出す。

## 決定

**B** を採用。`index_of(url_path)` をJinja2の `env.globals` に登録する。

## 理由

- テンプレート側で `index_of("/blog")[:3]` のようにJinja2のフィルター・スライスと
  自由に組み合わせられる
- 1ページで複数ディレクトリのスキャン結果を合わせて表示するような柔軟な使い方ができる
- フロントマターで宣言する方式はシンプルだが、「最新3件だけ」「カテゴリ絞り込み」などを
  表現するために結局テンプレート側のロジックが必要になる

## dateフィールドとドラフト

- `date:` フロントマターがないファイルはドラフトとして `index_of` の結果から除外される
- `date:` はYAMLが自動解釈できる形式のみサポート（`2026-04-25` 等）
- スラッシュ区切り（`2026/04/25`）は非サポート。実装を膨れさせてまで対応する価値がないと判断。
  入れてしまった場合はドラフト扱いになるだけでエラーにはならない
- タイムゾーンなしの日付はフロントマター/defaults.jsonの `timezone:` → システムTZで補完
