# ADR-006: LLMクローラーにはMarkdownを返す

## 状況

LLMがWebコンテンツを学習・参照する際、HTMLよりもMarkdownの方が
ノイズ（タグ・スタイル）が少なく処理しやすい。

## 決定

ClaudeBot・GPTBot・ChatGPT-User・PerplexityBot・meta-externalagent のUser-Agentを持つ
クローラーに対して以下の二段構えでMarkdownを提供する。

**1. ディレクトリアクセス時:** `index.md` を `text/markdown` でそのまま返す。HTMLレンダリングは行わない。

**2. `.html` アクセス時:** 対応する `.md` ファイルが存在する場合、その `.md` のURLへ302リダイレクトする。

本CMSではMarkdownファイル内のリンクは `.md` 拡張子のままであり、HTMLレンダリング時に
動的に `.html` へ書き換えている。そのため、一度 `.md` へ誘導されたクローラーは
以降 `.md` のリンクのみを辿ることになり、`.html` には戻らない。
302（一時リダイレクト）を使用するのは、`.html` のURLを永続的に `.md` として
正規化する意図ではなく、このUA向けのコンテキスト依存の振る舞いであるため。

また、MJ12bot のような積極的（Eager）なクローラーも同様にMarkdownを返す対象とする。
これらはLLMクローラーではないが、アクセス量が多く負荷になりやすいため、
HTMLよりも構造がシンプルなMarkdownを返すことでダウンロードサイズを減らし
トラフィックを抑制することを意図している。

**方針:** LLMクローラーは `LLM_CRAWLERS`、Eagerなボットは `EAGER_CRAWLERS` として
別々に定義し、判定時に連結する。

## 理由

- HTMLレンダリングはブラウザ向けの処理であり、LLMには不要
- Markdownのリンクは `.md` のままなので、クローラーにとって正確な情報
- GoogleBotは検索インデックスへの影響があるため対象外とした
- Eagerなボットへは軽量なレスポンスを返すことでサーバー負荷・帯域を節約できる

## トレードオフ

- User-Agentは偽装可能だが、意図的に偽装して `.md` を取得しようとする攻撃の実害はない
- `.html` アクセス時のリダイレクトは1往復のコストが発生するが、以降のクロールは `.md` のみとなるため許容できる
- `LLM_CRAWLERS` という名称は厳密にはLLM専用ではないが、主用途がLLMクローラー対応であるため据え置く
