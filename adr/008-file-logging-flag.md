# ADR-008: ファイルロギングの制御フラグ設計

## 状況

アクセスログの出力先として、uvicornの標準出力ログと `logs/access_log` ファイルへの書き込みの2系統がある。
当初は `--no-uvicorn-access-log` というフラグでuvicornのログを抑制し、その代わりにファイルへ書くという設計だった。

## 問題

- `--no-uvicorn-access-log` はuvicornログを「切る」という否定的な命名であり、本来の意図（ファイルロギングを有効にする）が伝わりにくかった。
- uvicornアクセスログが出ている状態（デフォルト）はローカルでのテスト中を意味する。その場合はファイルに書き込む必要はなく、むしろノイズになる。
- 本番・systemd運用時にはuvicornのログは不要で、ファイルロギングを有効にしたい。この2つは常にセットで切り替わる。

## 決定

- フラグ名を `--file-logging` に変更する。
- `--file-logging` を指定した場合のみ `logs/access_log` への書き込みを行い、uvicornアクセスログは抑制する。
- `--file-logging` を指定しない場合（デフォルト）はファイルへの書き込みを行わず、uvicornアクセスログを出力する。

これにより「何を有効にするか」という肯定的な表現になり、2つの動作の連動関係も自然に理解できる。

## ログローテーションの閾値

合わせて、アクセスの少ないvhostでは深夜ローテーションにより小さなファイルが大量に生成される問題を解決するため、
ファイルサイズが閾値（デフォルト1MB）未満の場合はローテーションをスキップする仕組みを導入した（`--log-rotation-min-bytes` で変更可能）。
